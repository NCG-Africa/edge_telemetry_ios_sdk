# Privacy and App Store

What EdgeRum touches, what it never touches, and how to satisfy App
Review.

## Overview

EdgeRum is designed to be App-Store-reviewable out of the box. It does
not use the IDFA, does not trigger the App Tracking Transparency
prompt, does not import `AdSupport`, and ships a privacy manifest that
declares every restricted-reason API it uses.

## Identifiers

EdgeRum owns three identifiers, all generated locally:

| Identifier   | Storage                                     | Notes |
|--------------|---------------------------------------------|-------|
| `device.id`  | Keychain (`com.edge.rum.identity`)          | `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`. UserDefaults fallback when Keychain writes fail. iOS clears Keychain on uninstall on modern installs â€” `device.id` rotates on reinstall in practice. |
| `session.id` | UserDefaults suite `com.edge.rum.session`   | Rotates after 30 minutes of inactivity or on cold start. |
| `user.id`    | UserDefaults suite `com.edge.rum.session`   | SDK-owned anonymous id. `EdgeRum.identify(_:)` attaches host-app `user.name` / `user.email` / `user.phone` as additional attributes but **does not** change this id. |

All three are 8 bytes of `SecRandomCopyBytes` entropy formatted
`prefix_<epochMs>_<16 hex>_ios`. They are never sent to advertising
networks; they are never persisted to iCloud Keychain.

## Restricted-reason APIs

The privacy manifest at `PrivacyInfo.xcprivacy` declares every
restricted-reason API the SDK calls:

| API category | Reason code | Why we use it |
|--------------|-------------|---------------|
| File timestamp APIs | `C617.1` | Offline queue file housekeeping |
| System boot time   | `35F9.1` | Uptime for `page_load` |
| Disk space         | `E174.1` | `device.freeStorage` context |
| UserDefaults       | `CA92.1` | Session, user, and consented-location persistence |

App Store Connect now validates the merged manifest of the host app
plus every SDK. The SDK ships its own manifest at
`Sources/EdgeRum/Resources/PrivacyInfo.xcprivacy`; Xcode picks it up
automatically for SwiftPM and CocoaPods consumers. XCFramework
consumers can copy the manifest into their app bundle or rely on
Xcode's merging during archive.

## IDFA / ATT

EdgeRum does **not**:

- Import `AdSupport` or read `ASIdentifierManager.advertisingIdentifier`.
- Trigger `ATTrackingManager.requestTrackingAuthorization`.
- Read the device serial number, MAC address, or vendor identifier
  intended for cross-app tracking.

The SDK's anonymous `device.id` and `user.id` are local random values
generated by `SecRandomCopyBytes`. They do not provide cross-app
linkability, and they survive only as long as the host app's Keychain
and UserDefaults containers do.

## What we send

Every event carries a flat `attributes` map of primitives only â€”
`String`, `Int`, `Double`, `Bool`. No URL path components are added
beyond what the host app generates through its own `URLSession` calls;
URL sanitisation hooks let consumers redact further.

The reference payload â€” every field that may appear on the wire â€” lives
in [`docs/payload-example.jsonc`](https://github.com/NCG-Africa/edge_telemetry_ios_sdk/blob/main/docs/payload-example.jsonc)
and [`docs/payload-schema.json`](https://github.com/NCG-Africa/edge_telemetry_ios_sdk/blob/main/docs/payload-schema.json)
in the repo root.

## Data retention and the offline queue

When the live transport cannot reach the backend, batches spill to
files under `Library/Caches/edge-rum/queue/` for later replay. The
cache directory is not backed up to iCloud and is purged by the OS
under storage pressure; the queue is capped at
``EdgeRumConfig/maxQueueSize`` (default 200 events) with oldest-file-
first overflow. Nothing else is persisted to disk by the SDK.

## Sample privacy disclosure

If the host app maintains its own privacy disclosure, the EdgeRum
section can read:

> *EdgeRum collects anonymous performance data â€” frame render times,
> network request timings, screen names, taps, lifecycle transitions,
> native crash reports, and connectivity changes â€” to help [host app]
> diagnose performance and stability issues. No advertising or
> cross-app tracking identifiers are collected, and no personal
> information is captured unless the host app explicitly calls
> `EdgeRum.identify(_:)` with values it has obtained with user
> consent. Data is sent to [host app's backend] under [host app's
> retention policy].*

Adjust to match your own retention, your own user-identity flow, and
your backend's hosting region.
